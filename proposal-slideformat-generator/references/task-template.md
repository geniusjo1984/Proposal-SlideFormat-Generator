# TASK.md Template

Use this template to derive a project-specific `TASK.md` from the user's RfP.

## Required Principles

- Build `TASK.md` only after confirming that the actual RfP is present in `01.Input_RfP`.
- Preserve the source section hierarchy unless the user requests a different structure.
- Preserve task-level bullets that the slide set must reflect.
- Keep all titles and lead messages traceable to the source.
- Treat this file as the slide-planning authority for the workspace.

## Template

```md
# TASK.md

## 과업 정리

- Source document: `[local RfP path]`
- Deck objective: `[proposal goal]`
- Slide planning principle: one main slide per task unless the user requests another mapping

## 전체 과업 프레임워크

### 1. 제안서 전체 논리 구조

- Section 1: `[section title]`
- Section 2: `[section title]`
- Section 3: `[section title]`

### 2. 전체 과업 프레임워크 장표 구성안

- Slide objective: show the overall project logic and task structure
- Required content:
  - overall section flow
  - section-level purpose
  - task-level grouping

### 3. 공통 장표 템플릿

- Title rule: message-driven lead title
- Top summary rule: one line that states what the slide proves or proposes
- Preferred body modules:
  - task objective
  - scope or analysis target
  - key execution actions
  - method or review approach
  - expected output or deliverable

### 4. 공통 장표 작성 지침

- Preserve required source bullets
- Use proposal-style wording
- Keep the body structured and scannable

## 과업별 슬라이드 상세 구성

### 과업 1. [RfP task name]

- Source section: `[section id and title]`
- Upper-context label: `[section/task notation]`
- Slide title draft: `[lead message title]`
- Lead message: `[1 sentence]`
- Task objective:
  - `[item]`
- Must-include source bullets:
  - `[bullet 1]`
  - `[bullet 2]`
- Recommended slide form:
  - `[table | process | matrix | framework | roadmap | stakeholder map]`
- Expected deliverable:
  - `[item]`
- Terminology rule:
  - `[preferred wording]`

### 과업 2. [RfP task name]

- Repeat the same structure for every task.
```

## Practical Rule

Do not pre-fill all tasks with invented content. Duplicate the block and populate it only from the actual RfP.
