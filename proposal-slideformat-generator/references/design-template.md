# DESIGN.md Template

Use this template as the fixed design authority for proposal-slide workspaces. Treat the visual rules below as the default standard, not as optional suggestions.

```md
# DESIGN.md

## Scope

This guide applies to slides created from the source document below and the user instruction provided in this session.

- Source document: `[set the local RfP path here]`
- Source rule: do not add claims, data, examples, or interpretations beyond the provided material unless the user explicitly asks

## Role

Act as a top-tier management consultant in the style of McKinsey, BCG, or the Korea Productivity Center. Plan and design slides that are highly professional, analytically structured, visually clean, and information-dense without becoming cluttered.

## Authoritative Design Direction

For this engagement, the design direction below is authoritative.

- Use a white background only
- Prefer the representative colors over auxiliary colors
- Keep slides minimal in wording and disciplined in composition
- Write slide titles as lead messages, not noun-only labels
- Maintain a formal, authoritative, analytical consulting-report tone

## Canvas And Layout

- Slide ratio is mandatory: A4 landscape
- Target page size: `297mm x 210mm`
- Point reference size for export and layout math: approximately `841.9pt x 595.3pt`
- Do not use 16:9 composition for this engagement
- Place a single-line title at the top
- Separate title and body with a clear divider line
- Organize content so the reasoning flows left-to-right or top-to-bottom
- Use thin, sharp borders and separators
- Favor structured layouts such as step blocks, flowcharts, tables, comparison matrices, and tightly organized content modules
- Select the slide form according to the message and evidence; do not treat tables, processes, criteria blocks, or cards as inherently superior to one another
- Avoid decorative chrome that does not improve comprehension
- Add a clear upper-context label when the slide belongs to a larger proposal section
- Preferred upper-context format: `1. 물품∙유통 구조 파악 I (1)` or an equivalent section-task notation placed above or near the main title area

## Color System

- Background: `rgb(255, 255, 255)` / `#FFFFFF`
- Text: `rgb(0, 0, 0)` / `#000000`
- Representative color 1: `rgb(29, 46, 79)` / `#1D2E4F`
- Representative color 2: `rgb(0, 109, 187)` / `#006DBB`
- Representative color 3: `rgb(3, 177, 231)` / `#03B1E7`
- Auxiliary color 1: `rgb(192, 0, 0)` / `#C00000`
- Auxiliary color 2: `rgb(215, 219, 223)` / `#D7DBDF`
- Auxiliary color 3: `rgb(244, 245, 246)` / `#F4F5F6`
- Auxiliary color 4: `rgb(48, 85, 140)` / `#30558C`
- Auxiliary color 5: `rgb(50, 120, 184)` / `#3278B8`
- Auxiliary color 6: `rgb(90, 90, 91)` / `#5A5A5B`

## Typography

- Primary title font: `KoPub돋움체 Bold`
- Primary body font: `KoPub돋움체 Medium`
- Supporting font: `KoPub돋움체 Light`
- Fallback title font: `Pretendard Bold`
- Fallback body font: `Pretendard Medium`
- Fallback supporting font: `Pretendard Regular`
- Keep type hierarchy crisp and restrained
- Prefer short, high-signal sentences and concise bullets
- Do not use overly expressive display typography
- All line breaks must occur at word boundaries
- Do not break a Korean word or phrase fragment across lines
- For Korean text, wrap by eojeol units only
- Example: write `나는` on one line and `학교에 간다` on the next line if needed, but never split as `나는 학` and `교에 간다`

## Content And Tone

- Tone: authoritative, analytical, structured, and board-grade
- Content should feel like a professional consulting deliverable, not marketing copy
- Use concrete slide text, bullets, table labels, and chart captions only when grounded in the provided source material
- Titles must be lead messages that state the takeaway directly
- Replace generic headings like `시장 규모` with message-driven titles such as `시장은 [요인]에 힘입어 연평균 15% 성장하고 있음` when that message is supported by the source

## Composition Rules

- One clear headline message per slide
- Minimize full sentences in the body
- Use bullets only when they improve scan speed
- Prefer tables, labeled blocks, process steps, and evidence layouts over paragraph-heavy compositions
- Make hierarchy visible through spacing, alignment, and linework before using fills or boxes
- Use representative colors first for emphasis, accents, and key lines
- Use auxiliary colors only as support for contrast, status, or secondary grouping

## Prohibitions

- Do not add content beyond the provided file and user instruction
- Do not change the background away from white
- Do not default to colorful startup-style layouts, gradients, soft shadows, or playful illustrations
- Do not use noun-only slide titles when a supported lead message can be written
- Do not overload slides with prose

## Execution Notes For Agents

- Treat this file as the default design guide for planning, slide generation, revision, and export preparation in this workspace
- The required slide ratio for this workspace is A4 landscape, and this overrides generic 16:9 defaults
- If project defaults, templates, or validators mention `720pt x 405pt`, do not treat that as the target ratio for this engagement
- If `KoPub돋움체` is unavailable in the runtime environment, use `Pretendard` as the required fallback font family instead of changing to another arbitrary font
```
