# DESIGN.md Template

Use this template to define the design authority for a proposal-slide workspace.

```md
# DESIGN.md

## Scope

This guide applies to slides created from the source document below and the user instruction provided in this session.

- Source document: `[set the local RfP path here]`
- Source rule: do not add claims, data, examples, or interpretations beyond the provided material unless the user explicitly asks

## Role

Act as a top-tier management consultant. Plan and design slides that are analytically structured, visually disciplined, and presentation-ready without becoming cluttered.

## Authoritative Design Direction

- Use a white background unless the user instructs otherwise
- Prefer restrained corporate or public-sector styling over startup-style layouts
- Write slide titles as lead messages, not noun-only labels
- Maintain a formal and proposal-ready tone

## Canvas And Layout

- Define the required slide ratio
- Define the target export size if relevant
- Place a single-line title at the top
- Separate the title area from the body with a clear divider or spacing rule
- Organize reasoning left-to-right or top-to-bottom
- Favor tables, matrices, process flows, criteria blocks, and structured modules over paragraphs
- Define the upper-context notation rule for section and task labels

## Color System

- Background: `[hex]`
- Text: `[hex]`
- Representative color 1: `[hex]`
- Representative color 2: `[hex]`
- Representative color 3: `[hex]`
- Auxiliary color 1: `[hex]`
- Auxiliary color 2: `[hex]`

## Typography

- Primary title font: `[font]`
- Primary body font: `[font]`
- Supporting font: `[font]`
- Fallback title font: `[font]`
- Fallback body font: `[font]`
- Fallback supporting font: `[font]`
- Define line-break rules for Korean or bilingual text if needed

## Content And Tone

- Tone: authoritative, analytical, and proposal-ready
- Prefer concise labels, bullets, and captions over prose
- Use only wording that is supported by the source material or explicit user instruction
- Titles must state the takeaway directly

## Composition Rules

- One clear headline message per slide
- Minimize full sentences in the body
- Use fills and accent colors sparingly
- Make hierarchy visible through spacing, alignment, and linework first

## Prohibitions

- Do not add unsupported facts or examples
- Do not overload slides with prose
- Do not use decorative elements that do not improve comprehension

## Execution Notes For Agents

- Treat this file as the design authority for planning, generation, revision, and export preparation
- If runtime fonts are missing, use the stated fallback fonts instead of switching arbitrarily
```
